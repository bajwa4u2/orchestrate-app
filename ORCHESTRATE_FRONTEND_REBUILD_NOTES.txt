Runtime contract
The backend uses header-based access context on protected operator and client endpoints.
Provide these values at runtime with --dart-define when needed.

Operator session
- API_BASE_URL
- OPERATOR_USER_ID
- OPERATOR_ORGANIZATION_ID
- OPERATOR_MEMBER_ROLE
- OPERATOR_USER_EMAIL (optional)

Client session
- API_BASE_URL
- CLIENT_USER_ID
- CLIENT_ORGANIZATION_ID
- CLIENT_ID
- CLIENT_USER_EMAIL (optional)

Example launch shape
flutter run -d chrome \
  --dart-define=API_BASE_URL=http://localhost:3000/v1 \
  --dart-define=OPERATOR_USER_ID=YOUR_USER_ID \
  --dart-define=OPERATOR_ORGANIZATION_ID=YOUR_ORG_ID \
  --dart-define=OPERATOR_MEMBER_ROLE=OWNER \
  --dart-define=CLIENT_USER_ID=YOUR_USER_ID \
  --dart-define=CLIENT_ORGANIZATION_ID=YOUR_ORG_ID \
  --dart-define=CLIENT_ID=YOUR_CLIENT_ID
